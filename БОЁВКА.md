# ⚔️ БУРМОЛДА — архитектура bullet-hell

← [РАЗРАБОТКА.md](РАЗРАБОТКА.md) (общий гайд и ссылки на остальные доки)

Ход врага — фаза уворота. Раньше был ОДИН общий пул мелких цветных кружков на
всех врагов; теперь снесён и заменён 3-слойной архитектурой (у каждого босса
свой уникальный, ни с кем не пересекающийся мувсет; мобы — на отдельной лёгкой
системе). Символов старого пула (`_pick_pool`, `MOB_STYLE`, `BOSS_OPENING`,
`SPEED_COL`, `PATTERN_NAMES`, `_mk`, `_spawn_pat`…) в коде больше НЕТ.

---

## Слой 1 — тулкит примитивов

`scripts/core/bullet_kit.gd` (`class_name BulletKit`, static-функции над
`arena`, как `Sprites` над `CanvasItem`). Низкоуровневые кирпичики, НЕ готовые атаки:

- **фигурные угрозы** `spawn_shape(arena, shape, pos, vel, opts)` — узнаваемый
  силуэт: `&"rect"` планка · `&"blade"` лезвие (от пивота, можно `spin`) ·
  `&"segment"` кусок стены · `&"spinner"` вертушка · `&"orb"` круглая пуля ·
  `&"cross"` летающий крест (у обоих круглый хитбокс по радиусу `half.x` —
  для моб-буллетхелла; у креста визуальные лучи длиннее хитбокса). opts:
  `size/angle/spin/accel/warn/safe/pierce/tint/life`. Коллизия остальных фигур
  — ориентированный прямоугольник vs ядро души.
- **силовые поля** `add_force(arena, kind, opts)`/`remove_force` — `&"wind"`
  (направленный порыв+`gust`), `&"vacuum"` (тянет к `point`), `&"turbulence"`
  (шум управления), `&"conveyor"`. Двигают САМО сердце (греби против).
- **тайминговые зоны** `add_hazard_zone` (телеграф→бьёт внутри rect),
  `add_safe_zone` (внутри цело, снаружи бьёт — после телеграфа).
- **управление ареной**: `set_corridor` (жёсткие стены-коридор), `set_blue_mode`
  (гравитация+прыжок ↑/Space), `move_box` (переезд/сжатие коробки).
- **достижимость** `clamp_step(prev, target, max_step)` — щель/безопасная точка
  не «телепортируется» (гарантия честного пути).

Примитивы сами: честный **телеграф** (угроза с `warn>0` безвредна и помечена),
централизованный урон (`arena._bullet_hit()`), авто-очистка, грейз/искры.

## Слой 2 — база атаки/кита

`scripts/battle/boss_attack.gd` (`BossAttack`: `start(arena)`/`update(arena,delta)`/
`done` + `name`/`rule`, показываемые в HUD — order-in-chaos сохранён) и
`scripts/battle/boss_kit.gd` (`BossKit`: `opening()` — срежиссированные первые
биты, `pick(stage, phase2)` — следующий бит из СВОИХ атак; эскалация во 2-й
фазе = докрутка своих механик, не чужие атаки).

## Слой 3 — конкретные боссы

`scripts/battle/boss_<key>.gd`, атаки — **вложенными классами** (уникальны по
теме, не пересекаются):
- `boss_kalitin.gd` (KalitinKit) — компьютерное железо: вентиляторы+ОЗУ,
  перегрев (сжатие), скачки напряжения, кабельный хлыст.
- `boss_tsizi.gd` (TsiziKit) — ветер/давление: порыв, вакуум-воронка,
  турбулентность, воздушный тоннель (коридор), мусор на ветру.
- `boss_zhizha.gd` (ZhizhaKit) — болото/кислота: наступающая кислота, щупальца,
  капли, поглощение (сжатие арены), волна тины (безопасная зона-разрыв).
- `boss_overseer.gd`/`boss_pekl_master.gd`/`boss_tm.gd` — боссы Адской Шахты
  (Надзиратель/Магистр/ТМ): плеть, штампы-по-расписанию, четыре глаза и т.п.

**Диспетчер** — в `battle_scene.gd` реестр `BOSS_KIT_PATHS` (boss_key → файл кита).

Добавить нового босса = новый файл `boss_<key>.gd` (`extends BossKit`) с атаками
из примитивов + строка в `BOSS_KIT_PATHS` + запись в `bosses.json`; тулкит
примитивов при этом НЕ меняется. Тюнинг арены — константы в шапке battle_scene
(`GRAV/JUMP_V/HIT_R_MIN/MAX/SOUL_SPEED`…). Пошаговый рецепт — в
[РЕЦЕПТЫ.md](РЕЦЕПТЫ.md). Бэклог идей — в [ИДЕИ.md](ИДЕИ.md).

## Мобы — лёгкая отдельная система

`scripts/core/mob_threats.gd` (`class_name MobThreats`): ОТДЕЛЬНАЯ система —
**быстрый плотный буллет-хелл**, но короткий (фаза ~3-5с). **25 паттернов** в
`ARCHETYPES`, `sequence(mob_key)` выбирает ОДИН **случайно из всего пула,
независимо от вида моба** (параметр `mob_key` не влияет на выбор — разнообразие
важнее тематической привязки; хочешь 26-й — допиши имя в `ARCHETYPES`, класс в
подходящий файл `mob_patterns/` и ветку в `_make()`). Сами атаки лежат в
`scripts/core/mob_patterns/*.gd` (по типу снаряда, не по виду моба —
mob_threats.gd остался тонким диспетчером, ~70 строк). Типы снарядов/угроз в пуле:

- **круглые пули** `&"orb"` (15 паттернов): spray/rain/spin/burst/gust/
  zigzag/pulse/converge/wall/drizzle/orbit/snake/corner/mines (+cross ниже);
- **летающие кресты** `&"cross"` (3): xrain/xspray/xspiral — вертящийся
  крест, круглый хитбокс меньше визуальных лучей (см. `BulletKit`);
- **тонкие быстрые лучи** (2): lasersweep/lasercross — те же `add_hazard_zone`,
  что у боссов, но тоньше (11px) и с коротким телеграфом (0.32-0.4с), чтобы
  уложиться в 3-5с бой;
- **самонаводящиеся ракеты со взрывным следом** (2): rockets/rocketpair —
  орб, наводится ~0.9с (атака сама рулит `vel` в `update()`), потом летит
  прямо, тянет искровой след (`arena._spark_burst`), в конце жизни взрывается
  телеграфированным кольцом;
- **почти-моментальные «снайперы»** (3): sniper/sniperduo/snipervolley —
  у края мигает точка-origin (это `warn`-телеграф орба: пуля стоит на месте,
  показывая ТОЛЬКО origin, но НЕ линию), потом очень быстрый (330px/с)
  выстрел по позиции, где игрок был на момент спавна — траекторию надо
  угадать и двигаться. SniperDuo/Volley наследуют `Sniper._fire()`.

Резко и динамично, но недолго (без комбо и эскалации). Мобы шустрые
(`speed_mult` мобов поднят до 1.05–1.3).

## Биом — только рамка/душа, никогда паттерн

**Биом влияет ТОЛЬКО на рамку/душу, никогда на выбор паттерна** (`_biome_kind()`
в `battle_scene.gd`): `"ice"` (лёд/опера) — душа скользит по инерции; `"hot"`
(вулкан/ад **и Адская Шахта** — `dungeon_scene.gd` ставит `biome="hell"` на все
бои шахты) — рамка МОБА сжимается на старте (~×0.72, только для `boss_key ==
null`, у боссов рамкой управляет их кит) + стенки жгут при касании (это уже было,
универсально для всех); `"goo"` (болото/вода/джунгли/кислота) — вязкое движение;
`"wind"` (пустоши `wastes`/`waste2`) — постоянное силовое поле `add_force(&"wind",
…)` сносит душу МОБА весь бой (тоже только `boss_key == null`). Эффект рамки/души
и случайный паттерн атаки — два независимых слоя, которые просто накладываются
в одном бою.

## Арена-хост (`battle_scene.gd`)

Держит состояние поля (`bullets/zones/forces/box/soul/corridor`), гоняет
жизненный цикл: `_init_fight_ai()` (в `_ready`) создаёт кит босса ОДИН раз на
бой — `_opening`/`_beat_i` живут сквозь ходы, поэтому opening проигрывается один
раз за бой, а дальше `pick(stage, phase2)` с настоящей эскалацией 2-й фазы
(иначе opening replay каждый ход маскировал бы фазу 2). `_begin_bullets`
(каждый ход) сбрасывает поле и берёт первый бит; `_bullets_step` двигает
душу+поля, зовёт `BulletKit.step_hazards`, прогоняет активные атаки, спавнит
стрей-пули боссов и берёт следующий бит; `_draw` делегирует в `BulletKit.draw_all`.
Фабрики-обёртки (`spawn_shape/add_force/…`), свойства-контракт
`arena.soul/box/phase2/stage/bh_t/warn_mult/speed_mult`. До 2 активных атак
одновременно (`_active_attacks`).

**Что сохранилось** поверх новой системы: динамический хитбокс (`_hit_r`,
плотнее шквал → меньше сердце), джус (`_shake/_hitstop/_hitflash/sparks/
_box_flash`), уворот `Battle.evade_chance()` (мультипликативный, потолок 60%,
деградация зелий), биом-физика (`_biome_kind`: лёд скользит, вулкан жжёт стены,
болото вязкое), драматургия по этапам (`stage`/`speed_mult`/`warn_mult`).

**Темп и длительность** (в `_begin_bullets`/`_bullets_step`): мобы — короткий
резкий буллет-хелл круглыми пулями, выжить ~3-5с (`_bh_dur = 3.4 + danger*0.1`).
Боссы — ДЛИННАЯ фаза (`_bh_dur = 14.0 + danger*0.24 + 3.2·phase2 + 0.9·stage`)
+ **арена-слой случайных «стрей»-пуль** (`_spawn_stray`, только боссы, не в
синем режиме): одиночная телеграфированная угроза с края (наводящаяся / лезвие
наискось / парная мелочь), частота растёт к финалу — даёт «резкость» и
разнообразие поверх авторских атак, не трогая киты.

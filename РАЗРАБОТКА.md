# 🧪 БУРМОЛДА — гайд разработчика (Godot 4.7)

Этот файл — подробная передача проекта. Его задача: чтобы новый разработчик
(и его ИИ-ассистент) за 15 минут понял, **как игра устроена**, **как её
запускать/тестировать** и **как добавлять контент, ничего не ломая**.
Сюжет, персонажи и связи квестов — в [СЮЖЕТ.md](СЮЖЕТ.md) (читай его перед
правками контента, чтобы понимать, о чём речь в данных).

> TL;DR: игра **data-driven**. Почти весь контент (предметы, локации, NPC,
> реплики, мобы, квесты, спрайты) лежит в `data/*.json`. Логика — в GDScript
> (`scripts/core`), показ — в сценах (`scripts/scenes`, `scenes/*.tscn`).
> Хочешь добавить меч/моба/локацию — обычно достаточно **отредактировать JSON**.

---

## 0. Что это за игра

«БУРМОЛДА» — шуточная кринж/свэг RPG про ядовитое болото, сигма-ведьму Ямполь,
плетунью лиан Шестухину, скелетона Калитина и духа Цизи. Стиль — как Undertale:
надмир с ходьбой по клеткам + пошаговый бой с фазой уворота (bullet-hell).

Это **перенос** оригинальной pygame-версии (лежит рядом в папке
`../burmolda-python`) на Godot 4. Перенос охватил фазы 0–4; **фаза 5 (экспорт в
.exe/браузер) намеренно НЕ сделана** — запуск пока через редактор Godot / `Играть.bat`.

## 1. Быстрый старт

### Запустить игру
- Двойной клик по **`Играть.bat`** (он открывает портативный Godot из `tools/godot/`
  и запускает проект), **или**
- открой `tools/godot/Godot_v4.7-stable_win64.exe` → **Import** → выбери
  `project.godot` → ▶ (**F5**).

**Управление:** стрелки — ходьба (можно зажать) и меню; **Enter/Space** — выбрать/дальше;
**Esc** — назад / быстрый сейв. Подойти к NPC-спрайту = поговорить; **войти в моба** = бой;
жёлтая клетка = переход; **Enter** в надмире = меню (Экипировка/Профиль/Квесты/Журнал/Сейв).

### Прогнать автотесты И аудит геймплея (headless, без окна)
```
tools\godot\Godot_v4.7-stable_win64_console.exe --headless --path . -- --test
tools\godot\Godot_v4.7-stable_win64_console.exe --headless --path . -- --audit
```
`--test` печатает `=== ВСЁ ЗЕЛЁНОЕ (N проверок) ===`, код возврата 1 при провале.
`--audit` (`tests/audit.gd`) «проигрывает» мир: BFS-проходимость всех локаций,
зачищаемость этажей шахты, выполнимость квестов, целостность опций NPC,
симметрию переходов — должен печатать `НАЙДЕНО ПРОБЛЕМ: 0`. **Гоняй ОБА после
правок контента** — аудит ловит битые данные, которые тесты не видят.

Чит-меню — **секретная клавиша F1** в надмире (работает всегда, но HUD о ней
не рассказывает): уровни, деньги, сеты, все приёмы, телепорт в любую локацию.

### После добавления НОВОГО .gd-скрипта с `class_name`
Один раз прогони импорт, чтобы Godot зарегистрировал глобальные классы:
```
tools\godot\Godot_v4.7-stable_win64_console.exe --headless --path . --import
```
(Иначе будут ошибки «Could not find type … in the current scope».)

## 2. Архитектура (главное, что нужно понять)

```
             ДАННЫЕ (data/*.json)  ←— источник истины, правится руками
                     │
                     ▼
   ЛОГИКА (scripts/core/*.gd)   ←— движко-независимые правила: бой, предметы,
     Player, Items, Battle,          квесты, лут, экономика, спрайт-рендер
     Attacks, Quests, Loot,
     Econ, NPCEffects, Sprites
                     │
                     ▼
   ПОКАЗ (scripts/scenes + scenes/*.tscn)  ←— сцены Godot: надмир, бой, NPC,
     overworld, battle, npc,                    инвентарь, кузнец, подземелье…
     inventory, dungeon, …
```

- **Autoload-синглтоны** (глобальные, доступны отовсюду по имени; заданы в `project.godot`):
  - `DataDB` — грузит весь `data/*.json` в память на старте.
  - `GameState` — держит текущего `player` и сейв/лоад (`user://save.json`).
  - `Sfx` — процедурные звуки.
- **Глобальные классы** (`class_name`, тоже доступны отовсюду): `Player`, `Items`,
  `Battle`, `Attacks`, `Quests`, `Loot`, `Econ`, `NPCEffects`, `Sprites`, `SoulMenu`.
- **Виртуальное разрешение** 640×480, стретч `canvas_items` (окно масштабируется), рендер
  GL Compatibility (максимальная совместимость + веб в будущем).

Почему так: логика отделена от движка (как и в оригинале), поэтому баланс/контент
менять легко и безопасно, а показ можно переделывать, не трогая правила.

## 4. Игровой поток (что открывает что)

```
Main.tscn (main.gd)
  └─ Title (title.gd)  --Enter-->  Overworld (overworld.gd)   ← центральный хаб
        ├─ войти в моба / 0.1% рандом ─────────► Battle (battle_scene.gd)
        │                                          └─ ход врага = BulletHell (внутри Battle)
        ├─ подойти к NPC ─────────► NPC (npc_scene.gd)
        │                             └─ квесты (Quests), покупки (NPCEffects), обучение приёмам
        ├─ подойти к боссу (K/Z) ─► Poster → Battle(boss)
        ├─ 'M' кузнец ────────────► Blacksmith
        ├─ 'X' вход в шахту ──────► Dungeon (20 комнат)
        ├─ нода m/f/h/j/c ────────► Gather   ·   нода e ──► Echo
        └─ Enter (меню) ──────────► Inventory / QuestLog / Journal / Профиль / Сейв
```

Сцены поверх надмира открываются как **оверлеи** в отдельном `CanvasLayer`
(флаг `busy` ставит ввод надмира на паузу), закрываются по сигналу `closed`/`done`/`battle_over`.

## 5. Форматы данных (data/*.json) — как редактировать контент

> Все JSON — **UTF-8, `ensure_ascii=false`** (кириллица читаемая). Godot грузит их
> в `DataDB` на старте. Числа без дробной части читаются как int.

### items/*.json — предметы
Разбиты по слотам. Каждый предмет:
```json
{ "id": "weapon_3_5", "name": "Легендарная катана", "slot": "weapon",
  "tier": 5, "color": "purple", "buffs": {"atk": 20},
  "buy": 980, "sell": 245, "base": 3 }
```
- `id` уникален. Схема id: `weapon_<база>_<тир>`, `armor_…`, `shield_…`, `trinket_…`;
  кольца — `ring_<эффект>_<тир>`.
- `slot`: `weapon|armor|shield|trinket|ring`.
- `tier` 0..5 (Ржавый→Легендарный, см. `tiers.json`). Оружие имеет тиры 0–5,
  броня/щит/оберег — 0–4, кольца — 1/3/5.
- `buffs`: любые из `atk, def, hp, swag, crit, block`.
- Кольца вместо `buffs` имеют `effect` + `mag` (см. `ring_effects.json`):
  `lifesteal, power, crit, poison, freeze, thorns`.
- Всего 118 предметов. Хочешь новый — просто добавь запись в нужный файл.

Служебное: `tiers.json` (префикс/множитель/цвет/цена по тиру), `slot_names.json`,
`stat_names.json`, `ring_effects.json` (id эффекта → {name, desc}).

### locations/<id>.json — локация
```json
{ "name": "Топкая Тропа", "biome": "swamp", "danger": 2,
  "map": ["########", "#..Y..1#", "########"],
  "entry": [3, 1],
  "npcs": {"Y": "yampol"}, "exits": {"1": "forest"},
  "boss": null, "wind": false, "fish": false,
  "monsters": [["Комар", 14, 3], ["Жаба-кринж", 22, 5]],
  "min_level": 1, "nodes": {"m": "mine"} }
```
**Тайлы карты** (`map` — массив строк одинаковой длины):
| символ | что | проходимо |
|---|---|---|
| `#` `T` `R` | стена / дерево / камень | нет |
| `~` | вода | нет |
| `i` `s` | лёд / снег | да |
| `,` `"` | трава / высокая трава | да |
| `.` (пробел) | земля/тропа | да |
| `1`–`9` | портал перехода (см. `exits`) | ступаешь → переход |
| заглавные буквы | NPC (см. `npcs`) и боссы `K`=Калитин, `Z`=Цизи | бамп → диалог/бой |
| `m f h j c e` | ноды: шахта/рыба/травы/жижа/кристалл/эхо | ступаешь → мини-игра |

- `danger` 0 = безопасная зона (мобы НЕ спавнятся). >0 = спавнятся видимые мобы
  из `monsters` (бродят; войти = бой). `min_level` — гейт: не пустит с низким уровнем.
- `monsters` — пул `[имя, базовое_HP, базовый_урон]`.
- `_index.json` — `{"start": "base", "order": [...]}` (стартовая локация и порядок).

### npcs/<kind>.json — персонаж
```json
{ "kind":"yampol", "name":"Ямполь", "color":"PURPLE",
  "flag":"met_yampol", "intro":"…", "gift":"оберег Ямполь",
  "lines":["…","…"],
  "options":[
    {"kind":"talk","label":"Поболтать","lines":["…"],"cringe":2,"swag_burst":false},
    {"kind":"buy","label":"Урок (-20)","cost":20,"effect":"gain_swag"},
    {"kind":"choice","label":"Ответить","prompt":"…","answers":[
        {"text":"…","line":"…","rep":2,"cringe":6,"money":10}]},
    {"kind":"learn","label":"✦ Приём (-40)","flag":"learn_sigma","move":"Сигма-взгляд","cost":40},
    {"kind":"sell","label":"Продать ресурсы"},
    {"kind":"leave","label":"Уйти"}
  ] }
```
- `color` — имя из палитры UI (PURPLE/TOXIC/SICK/CYAN/MAGENTA/RED/DEW).
- Эффекты покупок/кастомов заданы **строкой-ID** (`effect`), реализация — в
  `npc_effects.gd`. Доступные ID: `gain_swag, bless_hp, heal_full, sell_potion,
  evade_potion, feast, beef_up, give_item(+arg), give_food(+arg), ladushki,
  vozduhan_hint, vozduhan_bet, vozduhan_quest_take/truth/lie`.
- У любой опции могут быть условия видимости: `req_flag` (флаг должен стоять)
  и/или `req_not_flag` (не должен) — фильтруются в `npc_scene._rebuild_options`.
- Вид опции `fetch` — «принеси предметы»: `items {имя:кол-во}` + `money`/`cringe`
  /`line`; проверяет инвентарь, забирает, платит (повторяемо).
- **Блуждающие NPC** (без буквы на карте): спавнятся в `overworld._spawn_wanderers`
  — Тепличная (гарантированно на базе при первом входе, дальше 1% в любой
  локации) и Зомби (5% в `ZOMBIE_SPOTS`). Бамп в них открывает обычный диалог.
- Эффект кольца `calm` (Кольцо Невозмутимости, `ring_calm_q`) добавляется к
  шансу уворота в `Battle.evade_chance()`.
- Спрайт NPC берётся по `kind` (см. `sprites.json`); если нет — по `npc_override`.

### monsters/, phrases/, quests/, прочее
- `monsters/bosses.json`: `{ "kalitin": ["Калитин", hp, dmg, "art", ["таунт", …]] }`.
- `monsters/enemies.json`: общий пул `[[имя,hp,dmg], …]`.
- `monsters/mob_attacks.json`: строки-реплики врага, плейсхолдер `{name}`.
- `monsters/food.json`: `[[имя, лечение, цена], …]` (еда/лечилки в бою).
- `phrases/<bank>.json`: массив строк. Доступ: `DataDB.phrase("BANK")`
  (например `BURMOLZH`, `SWAMP_FACTS`, `SWAG_BURSTS`, `KALITIN_FEAR`, …).
- `quests/quests.json`: см. §6 (Quests).
- `resources.json`: `{ "ржавая руда": 5, … }` (цена продажи).
- `balance/combat.json`: `START_*`, `LEVEL_STEP`, `RANKS`, `BOSS_HP_MULT`, `ATTACK_MISS`.
- `sprites.json`: `{pal, grids, mob_keywords, npc_override}` (см. §7).

## 6. Ядровые системы (scripts/core)

### Player (`player.gd`)
Статы: `burmolda` (валюта), `cringe` (опыт), `swag` (сила), `level`, `hp/max_hp`,
`inventory` (имя→кол-во), `equipment` (6 слотов), `reputation`, `flags`, `quests`, `stats`.
`add_cringe()` копит опыт и повышает уровень (`LEVEL_STEP` из balance). Сейв —
`to_dict()/from_dict()` → JSON.

### Items (`items.gd`)
Данные — в `DataDB.items` (из JSON). Слоты: `weapon, armor, shield, trinket, ring1, ring2`.
Функции: `equip/unequip`, `total_buffs(player)` (сумма статов), `ring_effects(player)`
(эффекты колец), `random_item(max_tier, slot)`, `buffs_str(id)`.

### Battle (`battle.gd`) — модель боя, формулы 1:1 с оригиналом
- `attack_power = swag*2 + level + atk-бафф`.
- Крит: `0.12 + (crit-оберег + crit-кольцо)/100`. Промах: `ATTACK_MISS` (0.08).
- `_hit(dmg)` применяет эффекты колец: `power`(+урон), `lifesteal`(хил), `poison`
  (стаки → `poison_tick()` в ход врага), `freeze`(шанс `enemy_frozen` — враг пропускает ход).
- **Сложность**: конструктор принимает `danger` локации — мобы получают +9% HP и
  +5% урона за каждый пункт опасности выше 2; награды растут (`+danger*6` бурмолды,
  `+danger*3` кринжа). Боссы: HP × `BOSS_HP_MULT` (1.6–1.9) + `level*6` — бой 8–12 ходов.
- Исходы: `resolve_victory()` (награда + трофей + `Quests.on_kill` + левелап),
  `resolve_defeat()` (hp=1, «болото не отпускает»), `flee()`.

### Bullet-hell паттерны (`battle_scene.gd`)
Фаза уворота — 10 паттернов: `aimed` (рой, иногда веером из трёх), `rain`
(ливень), `walls` (кости с проходом, 0.25с мигают-телеграфят), `spiral` (вихрь;
в ярости — 3 рукава), `homing` (доводят курс ~1.15с, потом летят по прямой),
`laser` (резкий телеграф 0.45с → луч), `wind` (ветер сносит душу), `gapwall`
(«забор» — стена сверху с дрейфующей щелью), `burst` («залпы» — 2-3 точки
одновременно выпускают кольца, снаряды 0.32с «проклёвываются» и лишь потом
летят), `gravity` (**СИНЯЯ ДУША**: гравитация + прыжок ↑/Space, кости едут по
полу или свисают с потолка — режим Санса; эксклюзивен, не комбинируется).
Механика:
- **Комбо и смена**: внутри фазы каждые ~2.5с набор атак перебрасывается
  (`_reshuffle`), с шансом активны **2 паттерна одновременно** (`active`).
- **Пул** (`_pick_pool`): у боссов фирменный (Калитин — кости/лучи, Цизи — ветер/
  вихрь, Жижа — ливень/орбы; во 2-й фазе добавляются gapwall/burst/gravity),
  у мобов — **по виду** (`MOB_STYLE` по ключу спрайта); danger 6+ добавляет орбы/лучи.
- **Этапы боя** (`_stage`): бой босса — драматургия по ходам (`turn_no`):
  0 интро (одиночные паттерны, медленнее) → 1 разгон (комбо 50%) → 2 фаза
  ярости (≤50% HP, `Battle.check_phase2`: +2 урона, комбо 75%) → 3 финал
  (с 8-го хода в ярости: комбо 95%, максимум скорости). Скорость снарядов
  ×1.05→×1.34 по этапам (`_speed_mult`); мобы наоборот мягче (×0.88–1.12 от
  danger) — разрыв мобы/боссы заметен. **Открытия боссов срежиссированы**:
  первые сегменты идут по списку `BOSS_OPENING` (авторская подача паттернов),
  дальше — случайная эскалация; фаза 2 прерывает сценарий.
- **Интенсивность без простоев**: фоновая «морось» (`_drizzle_cd`) — медленный
  шар в душу даже между волнами (отучает стоять в углу); разовый «спайк»
  (`_spike_at`) — внезапная очередь в середине сегмента; сегменты короткие
  (1.7–2.4с) — за фазу 2–4 разных «бита». Темп растёт и ВНУТРИ фазы: скорость
  до +15% (`_bh_t` в `_speed_mult`), спавн до −12% кулдаунов (`_rate`);
  телеграфы поджимаются в этапах 2–3 (`_warn_mult` ×0.85/×0.72 — лучи, кости,
  залпы). Повторы паттернов мутируют (`_pat_seen`): проход стены костей
  сужается, шанс веера роя растёт, вихрь меняет направление (`_spin`).
- **Динамический хитбокс** (`_hit_r`): плотный шквал (много пуль) плавно
  сжимает хитбокс 6.8→4.4 (`HIT_R_MIN/MAX`) — щели проходимы честно; сердце
  визуально сжимается вместе с ним + белое свечение-телеграф.
- **Гарантия проходимости** (правило: каждый паттерн решаем без урона):
  КОСТИ и ЗАБОР не комбинируются (не запечатывают путь); проход новой стены
  костей ≤55 пкс от прошлого (`_last_gap_y`); щель забора дрейфует ≤58 пкс
  за ряд (`_gap_x`); в синем режиме при висящей потолочной кости низовые —
  только перепрыгиваемые, вторая потолочная не спавнится.
- **Цвет пули = скорость**: голубая ×0.68, жёлтая ×1, красная ×1.38 (`_mk`, `cls`).
- **🟢 Зелёные обманки**: ~16% круглых снарядов безвредны (флаг `safe`).
- **🔵/🟠 Пули-правила** (`beh`, у боссов и danger 4+): синяя не бьёт СТОЯЩЕГО,
  оранжевая не бьёт ДВИЖУЩЕГОСЯ (порог 14 пкс/с, `_beh_blocks`) — как у
  Асгора/Андайн. Тег-подсказка в легенде: «🔵 замри · 🟠 беги».
- **Честные хитбоксы**: ядро души 6×6 (сердечко рисуется 16), пуля бьёт при
  дистанции < `HIT_R`=6 — заметно меньше спрайтов, впритирку уворачиваться честно.
- **Грейз**: пуля прошла ближе 13 пкс — искры, кольцо, лёгкая дрожь (`_graze_t`).
- **Джус**: стоп-кадр `_hitstop` 0.08с + белая вспышка `_hitflash` при уроне и
  смене фазы, искры-частицы (`sparks`, `_spark_burst`) на попадание/грейз/выстрел
  лазера, тряска (`_shake`).
- **Коробка живёт**: меняет размер И место (`box_target`), переезд быстрый
  (lerp ×9) с вспышкой-телеграфом `_box_flash`; в синем режиме — низкий коридор.
- **Уворот**: `Battle.evade_chance()` — источники (Блок%, кольцо `calm`, каждое
  зелье уворота) стакаются **мультипликативно** (1−Π(1−s), как в Доте), жёсткий
  потолок 60%. Зелья и роса **деградируют в течение боя** (×0.6 за повторный
  приём, `_potion_mult`), в новом бою эффективность восстанавливается.
- **Биом-эффекты** (`_biome_kind`, биом передаёт надмир через `b.biome`; в
  подземелье пусто → без эффекта): лёд/опера — душа скользит по инерции;
  вулкан/hell — стенки коробки жгут при касании (в синем режиме пол — опора,
  не жжёт); болото/вода/джунгли/кислота — вязко (−22% скорости). Рамка коробки
  перекрашивается, тег в подсказке.
Новый паттерн = спавнер в `_spawn_pat()` + движение/отрисовка по `kind`.
Тюнинг — константы в шапке файла (`LASER_WARN`, `HIT_R`, `GRAV`, `JUMP_V`…).
Бэклог идей боёвки — в `ИДЕИ.md`.

### Плакаты боссов (`poster_scene.gd`)
Полноэкранный пиксель-постер перед боссом: портрет из пиксель-сетки
(`sprite_key`), вращающиеся лучи, ореол, пиксельная рамка, титул + строка-лор
(`flavor`). Конфиги надмирных боссов — `overworld.gd → BOSS_POSTER`
(kalitin/tsizi/zhizha), ТМ — в `dungeon_scene._tm_poster`. Утверждено владельцем.

### Attacks (`attacks.gd`)
`available(player)` строит меню БОЯ по снаряге и флагам:
- всегда: `basic` (Бурмольнуть), `roar` (Сигма-рёв, риск);
- по тиру оружия: `cleave`(≥1), `flurry`(≥3), `legend`(≥5);
- при щите: `shieldbash`;
- выученные у NPC (флаги `learn_*`): `sigma_gaze, bogatyr, ice_aria, crystal`.
`execute(id, battle)` выполняет приём.

### Quests (`quests.gd`)
`QUESTS` — в `data/quests/quests.json`. Тип `kill` (по ключевым словам в имени
моба, `count` штук) или `boss` (по флагу победы). `givable/ready` — что NPC может
выдать/принять; `start/turn_in`; `on_kill` вызывается из `Battle.resolve_victory`.
`main_goal(player)` — подсказка-цель. NPC-диалог сам подмешивает пункты «Взять/Сдать».

### Loot (`loot.gd`), Econ (`econ.gd`), NPCEffects (`npc_effects.gd`)
- Loot: `grant(player, danger, is_boss)` — ресурс + шанс на снаряжение (тир растёт с danger).
- Econ: `sell_all(player)` — продать все ресурсы (цены из `resources.json`).
- NPCEffects: `apply(player, effect, arg)` и `buy(player, cost, effect, arg)` — по effect-ID.

## 7. Спрайты — пиксель-арт 16×16 (важно!)

Спрайтов-картинок нет: всё рисуется **процедурно из сеток**. Данные — `data/sprites.json`:
- `pal`: символ → `[r,g,b]` (палитра; `.` и пробел = прозрачно).
- `grids`: ключ → массив из 16 строк по 16 символов (сам пиксель-арт).
- `mob_keywords`: `[["комар","komar"], …]` — по подстроке имени подбирается ключ.
- `npc_override`: `{"fish_npc":"fish"}` — если ключ NPC ≠ его `kind`.

Рендер — `Sprites.draw_grid(canvas, rect, key)` (рисует прямо в `_draw`,
nearest-neighbour). Хелперы: `Sprites.mob_key(name)`, `Sprites.npc_key(kind)`,
`Sprites.draw_mob/ draw_npc`, `Sprites.draw_heart(...)` (душа-сердечко).

Ключи спрайтов приходят из оригинала (46 штук) + **доп. сетки заданы прямо в коде**
в `data_db.gd → _inject_extra_sprites()` (`smith`, `dungeon`). Там же — образец, как
руками нарисовать новую сетку символами палитры.

## 8. Как добавить контент (рецепты)

- **Новый меч/предмет:** добавь запись в `data/items/swords.json` (или нужный слот).
  Уникальный `id`, `slot`, `tier`, `buffs`, `buy`, `sell`. Всё — заработает само.
- **Новое кольцо-эффект:** эффекты — в `ring_effects.json`; сами кольца — в `rings.json`
  (`effect`+`mag`). Новый ТИП эффекта требует правки `Battle._hit`/`poison_tick` в `battle.gd`.
- **Новый моб:** добавь `[имя, hp, dmg]` в `monsters` нужной локации (или в `enemies.json`).
  Спрайт подберётся по ключевому слову; хочешь свой — добавь пару в `sprites.json → mob_keywords`
  и сетку в `grids`.
- **Новая локация:** создай `data/locations/<id>.json` (см. §5) и впиши `<id>` в
  `locations/_index.json → order`. Свяжи порталом: у соседа в `exits` добавь `"цифра":"<id>"`,
  а на его карте поставь эту цифру; и наоборот.
- **Новый NPC:** `data/npcs/<kind>.json` + `<kind>` в `npcs/_index.json`. Поставь букву NPC
  на карту локации и в её `npcs` (`"Q":"<kind>"`). Спрайт — ключ `<kind>` в `sprites.json`.
- **Новый квест:** запись в `data/quests/quests.json` (`giver/turnin/type/keys/count/reward/requires`).
- **Новый приём боя:** реализуй `static func` в `attacks.gd`, добавь в `available()` (условие)
  и в `execute()` (диспетч). Если приём учат у NPC — заведи флаг `learn_x` и опцию `learn` у NPC.
- **Новый звук:** добавь тон в `sfx.gd → _bank`, вызывай `Sfx.play("имя")`.

После правки только JSON перезапуск import не нужен. После нового `.gd` с `class_name` —
прогони `--import` один раз (§1).

## 9. Соглашения и ГРАБЛИ (обязательно прочесть перед правкой кода)

- **GDScript и `:=` от Variant.** Значения из `Dictionary.get()`, индексов массива,
  `min/max` — типа `Variant`. `var x := variant_value` → ошибка «Cannot infer type…»
  (в 4.7 это ошибка). **Пиши явный тип:** `var x: int = ...`, или используй `mini/maxi/absi/absf`.
- **Фон сцены — рисуй в `_draw`, НЕ дочерним `ColorRect`.** В Godot дети рисуются
  ПОВЕРХ `_draw` родителя, поэтому полноэкранный `ColorRect`-ребёнок перекроет всё,
  что рисуешь в `_draw` (враг, пули, тайлы). Во всех сценах с `_draw` фон закрашивается
  первым прямо в `_draw`.
- **Не называй методы `_set`, `_get`, `_init` не по назначению** — это виртуальные
  методы Object. Наш сеттер клетки в подземелье называется `_put`, а не `_set`.
- **Headless НЕ рисует.** `--headless` не вызывает `_draw`, окна нет. Логику тестируем
  headless; визуал — только в окне (или скриншотером §11).
- **`--quit-after N`** добавляй к отладочным запускам, чтобы процесс сам завершался и не
  завис (особенно если сцена упала на компиляции и не дошла до `quit()`).
- **Launcher.** В `.bat` путь в `--path` НЕ должен кончаться на `\` в кавычках
  (`"...\NewBurmolda\"` → Windows съедает кавычку → «Invalid project path»). Поэтому
  `Играть.bat` использует `%CD%` (без хвостового слэша), а не `%~dp0`, для самого
  вызова Godot. `tools/godot/` в `.gitignore` (движок туда не коммитится — 170+ МБ
  бинарник), поэтому `Играть.bat` **сам скачивает** портативный Godot 4.7 при первом
  запуске (PowerShell `Invoke-WebRequest` + `Expand-Archive` с официального релиза
  GitHub), если `tools\godot\Godot_v4.7-stable_win64.exe` отсутствует — это и чинит
  запуск «на любом ПК» (у второго разработчика после клонирования движка нет).
  Содержимое `.bat` держим **строго ASCII** (без кириллицы) — раньше кириллица вместе
  с `chcp 65001` ломала парсинг на некоторых локалях Windows.
- **Эмодзи в тексте.** Дефолтный шрифт Godot не содержит цветных эмодзи → были квадраты.
  Починено фолбэком: `DataDB._setup_font()` добавляет `assets/fonts/NotoEmoji-Regular.ttf`
  в `ThemeDB.fallback_font.fallbacks`. Новый эмодзи не рисуется — проверь, есть ли он в шрифте.
- **Не запускай игровое ОКНО без спроса у владельца ПК.** Однажды оконный запуск во
  время его игры в полный экран подвесил комп. Для проверки логики — только headless.

## 11. Скриншотер (`tests/shot.gd`) — как «увидеть» игру без рук

`Main` по флагу `--shot` грузит `Shot.tscn`, который рендерит ключевые экраны в PNG
(в `user://` или заданную папку) и выходит. Запуск — **оконный** (headless не рисует):
```
tools\godot\Godot_v4.7-stable_win64_console.exe --path . -- --shot
```
Полезно, чтобы ИИ мог посмотреть на результат правок вёрстки/спрайтов. ⚠️ Открывает
окно — не запускай, если на этом ПК идёт полноэкранная игра. Это чисто отладочный
инструмент, в геймплее не участвует (срабатывает только по флагу).

## 12. Связь со старым проектом и сидер (`tools/seed.py`)

Данные `data/*.json` изначально **сгенерированы** из pygame-версии (`../burmolda-python`)
скриптом `tools/seed.py` (Python 3, только stdlib). Он импортирует движко-независимые
модули оригинала (предметы, локации, NPC, реплики, мобы, спрайты) и дампит в JSON.

Сейчас **источник истины — сам JSON** (правь его). Сидер нужен, только если захочешь
что-то заново вытащить из оригинала:
```
python tools\seed.py
```
⚠️ Перезапуск сидера **перезапишет** `data/*.json` — потеряешь ручные правки. Обычно он
больше не нужен.

## 14. Как эффективно вести разработку с ИИ

- Проси **сначала данные, потом код**: большинство фич = правка JSON.
- **Всегда проси прогонять headless-тесты** после изменений (`-- --test`) и добавлять
  новые проверки к новым системам.
- Напоминай про **грабли §9** (Variant `:=`, фон в `_draw`, `--quit-after`).
- Для проверки **визуала** — скриншотер (§11), но предупреди: **окно, не во время игры**.
- Держи изменения маленькими и проверяемыми; коммить рабочие состояния.

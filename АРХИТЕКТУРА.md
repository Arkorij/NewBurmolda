## Архитектура (главное, что нужно понять)

```
             ДАННЫЕ (data/*.json)  ←— источник истины, правится руками
                     │
                     ▼
   ЛОГИКА (scripts/core/*.gd)   ←— движко-независимые правила: бой, предметы,
     Player, Items, Battle,          квесты, лут, экономика, спрайт-рендер
     Attacks, Quests, Loot,
     Econ, NPCEffects, Sprites
                     │
                     ▼
   ПОКАЗ (scripts/scenes + scenes/*.tscn)  ←— сцены Godot: надмир, бой, NPC,
     overworld, battle, npc,                    инвентарь, кузнец, подземелье…
     inventory, dungeon, …
```

- **Autoload-синглтоны** (глобальные, доступны отовсюду по имени; заданы в `project.godot`):
  - `DataDB` — грузит весь `data/*.json` в память на старте.
  - `GameState` — держит текущего `player` и сейв/лоад (`user://save.json`).
  - `Sfx` — процедурные звуки.
- **Глобальные классы** (`class_name`, тоже доступны отовсюду): `Player`, `Items`,
  `Battle`, `Attacks`, `Quests`, `Loot`, `Econ`, `NPCEffects`, `Sprites`, `SoulMenu`,
  `HudPanel`, `UiText`.
- **Виртуальное разрешение** 640×480, стретч `canvas_items` (окно масштабируется), рендер
  GL Compatibility (максимальная совместимость + веб в будущем).

Почему так: логика отделена от движка (как и в оригинале), поэтому баланс/контент
менять легко и безопасно, а показ можно переделывать, не трогая правила.

## Структура проекта (что где лежит)

```
NewBurmolda/
  project.godot            конфиг Godot: главная сцена, автолоады, окно, рендер
  Играть.bat               запуск игры (использует %CD% — см. ГРАБЛИ.md)
  README.md                краткое описание
  РАЗРАБОТКА.md            общий гайд + ссылки на детальные доки (этот файл — одна из них)

  data/                    ★ ИСТОЧНИК ИСТИНЫ (см. ФОРМАТЫ.md)
    items/  swords.json armor.json shields.json trinkets.json rings.json
            tiers.json slot_names.json stat_names.json ring_effects.json
    locations/  <id>.json (по файлу на локацию) + _index.json
    npcs/       <kind>.json (по файлу на NPC) + _index.json
    phrases/    <bank>.json (банки реплик, имена в нижнем регистре)
    monsters/   biomes.json bosses.json enemies.json mob_attacks.json food.json
    quests/     quests.json
    balance/    combat.json (стартовые статы, множители боссов, шанс промаха, ранги)
    resources.json          ресурс -> цена продажи
    nodes.json              символ ноды -> тип; тип -> {title, emoji, resources}
    sprites.json            пиксель-арт: палитра + сетки 16x16 + маппинг имён

  scripts/core/            ЛОГИКА (движко-независимая)
    data_db.gd     autoload: грузит все JSON; тут же доп. спрайты (smith/dungeon)
    game_state.gd  autoload: текущий игрок, сейв/лоад
    player.gd      class Player: статы, инвентарь, экипировка, флаги, прокачка
    items.gd       class Items: экипировка/баффы/кольца (данные — в data/items)
    battle.gd      class Battle: вся модель боя (формулы 1:1 с оригиналом)
    attacks.gd     class Attacks: варианты приёмов + список доступных
    quests.gd      class Quests: выдать/трекать/сдать
    loot.gd        class Loot: трофеи после боя
    econ.gd        class Econ: продажа ресурсов
    npc_effects.gd class NPCEffects: эффекты NPC по effect-ID
    sprites.gd     class Sprites: рендер пиксель-сеток + сердечко + маппинг имён
    bullet_kit.gd  class BulletKit: тулкит примитивов буллет-хелла (см. БОЁВКА.md)
    mob_threats.gd class MobThreats: диспетчер угроз рядовых мобов — тонкий,
                   сами паттерны вынесены в mob_patterns/ (см. ниже)
    sfx.gd         autoload Sfx: мягкие процедурные звуки

  scripts/core/mob_patterns/   ПАТТЕРНЫ МОБОВ, по типу снаряда (см. БОЁВКА.md)
    orb_patterns.gd    15 паттернов, круглые пули `&"orb"`
    cross_patterns.gd  3 паттерна, летающие кресты `&"cross"`
    laser_patterns.gd  2 паттерна, тонкие быстрые лучи
    rocket_patterns.gd 2 паттерна, самонаводящиеся ракеты
    sniper_patterns.gd 3 паттерна, почти-моментальные пули с мигающим origin
    (файлы без class_name — просто «неймспейсы» с вложенными классами,
    загружаются через preload() в mob_threats.gd; не путать с scripts/battle/,
    где каждый файл — авторский кит КОНКРЕТНОГО босса, не общий пул)

  scripts/battle/          АВТОРСКИЕ АТАКИ ВРАГОВ (см. БОЁВКА.md)
    boss_attack.gd class BossAttack: один автомат-атака (start/update/done)
    boss_kit.gd    class BossKit: база кита (opening/pick)
    boss_kalitin.gd/…/boss_tm.gd  6 китов боссов, атаки — вложенными классами

  scripts/scenes/          ПОКАЗ (по скрипту на сцену)
    main.gd        загрузчик: роутит в Title (или тесты/скриншоты по флагу)
    title.gd       титульный экран
    overworld.gd   ★ надмир: карта, ходьба, видимые мобы, порталы, NPC, меню
    overworld_debug.gd  class OverworldDebug: чит-меню F1 + дебаг-бой F2,
                        вынесены из overworld.gd (тот же приём, что Sprites/
                        BulletKit — static-функции, первый аргумент `ow`)
    battle_scene.gd★ бой: меню, приёмы, арена-хост bullet-hell, исходы
    npc_scene.gd   диалог с NPC: портрет, реплики, опции, квесты
    inventory_scene.gd  инвентарь: слева ячейки надетого (6 слотов), справа
                        вкладки (оружие/броня/щиты/обереги/кольца/припасы/
                        добыча/квест) со скроллом; спрайты предметов, тир-цвета
    blacksmith_scene.gd кузнец (купить дорого / продать дёшево)
    dungeon_scene.gd    Адская Шахта: БЕСКОНЕЧНАЯ; палитры меняются по ярусам
                        глубины; комнаты: завалы/лава (реки, озёра с рыбами-
                        стрелками)/ВОДЯНЫЕ озёра/кристальные пещеры/грибные
                        гроты/костяные залы/пепельные зоны; расписание в
                        20-этажном цикле: стражник на 5 и 11/13/17/19 (больно,
                        но коротко), мини-босс на 10 и 15, «ТМ» на 20; после ТМ
                        — «ПЕКЛО II+»: пепельные твари жирнее/злее и их больше,
                        зато сундуков больше и редкий лут чаще; сундуки: с 10
                        этажа шанс двойных, 5%+ золотой (жирный лут); правая
                        дверь открывается после зачистки, назад — свободно,
                        выход = сброс этажей (лут остаётся); лор в LORE_*
    gather_scene.gd     мини-игра добычи (качающийся маркер)
    echo_scene.gd       мини-игра «Эхо болота» (повтори последовательность)
    poster_scene.gd     драматичный постер перед боссом
    questlog_scene.gd   журнал квестов
    journal_scene.gd    журнал болота (факты)

  scripts/ui/
    soul_menu.gd   class SoulMenu: меню в стиле Undertale (курсор-сердечко);
                   длинные списки скроллятся окном max_visible (▲/▼-индикаторы)
    hud.gd         class HudPanel: верхняя панель надмира — полосы HP и кринжа
                   (опыта), бурмолда, свэг, локация, тост-сообщения
    minimap.gd     class Minimap: миникарта локации в правом нижнем углу
                   (стены/вода/порталы/NPC/мобы/игрок) — пара с зумом камеры
                   (Overworld.CAM_ZOOM = 2.0, локации скроллятся за героем)
    ui_text.gd     class UiText: paginate() — режет длинные сообщения на
                   страницы (▼ ENTER), чтобы текст не налезал на меню/статы

  scenes/*.tscn            «тонкие» сцены: корневой узел + прикреплённый скрипт
  tests/                   headless-тесты (см. ТЕСТИРОВАНИЕ.md) + скриншотер
  tools/                   seed.py (см. ТЕСТИРОВАНИЕ.md) и портативный Godot 4.7
```

## Игровой поток (что открывает что)

```
Main.tscn (main.gd)
  └─ Title (title.gd)  --Enter-->  Overworld (overworld.gd)   ← центральный хаб
        ├─ войти в моба / 0.1% рандом ─────────► Battle (battle_scene.gd)
        │                                          └─ ход врага = BulletHell (внутри Battle)
        ├─ подойти к NPC ─────────► NPC (npc_scene.gd)
        │                             └─ квесты (Quests), покупки (NPCEffects), обучение приёмам
        ├─ подойти к боссу (K/Z) ─► Poster → Battle(boss)
        ├─ 'M' кузнец ────────────► Blacksmith
        ├─ 'X' вход в шахту ──────► Dungeon (20 комнат)
        ├─ нода m/f/h/j/c ────────► Gather   ·   нода e ──► Echo
        └─ Enter (меню) ──────────► Inventory / QuestLog / Journal / Профиль / Сейв
```

Сцены поверх надмира открываются как **оверлеи** в отдельном `CanvasLayer`
(флаг `busy` ставит ввод надмира на паузу), закрываются по сигналу `closed`/`done`/`battle_over`.

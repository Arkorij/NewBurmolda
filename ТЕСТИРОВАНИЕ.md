## Тесты (`tests/run_tests.gd`)

Запуск — headless:
```
tools\godot\Godot_v4.7-stable_win64_console.exe --headless --path . -- --test
```
Как работает: `Main` по флагу `--test` грузит `TestRunner.tscn`, тот в `_ready` гоняет
проверки, печатает `[OK]/[FAIL]` и выходит с кодом (1 при провале). Покрыто: целостность
данных (122 предмета / 35 локаций / 17 NPC / переходы резолвятся), математика экипировки,
меню приёмов, эффекты колец, TTK, квесты, примитивы буллет-хелла, авторские киты боссов,
лёгкая моб-система, диалоги (одноразовость/цепочки/гейты по уровню), инстанс всех сцен,
спрайты/шрифт/звук.

**Добавляя систему — добавь проверку** в соответствующую `_test_*` функцию (шаблон:
`check(условие, "описание")`). Это дешёвая страховка от регрессий, гоняется за секунды
и без окна.

## Аудит геймплея (`tests/audit.gd`)

```
tools\godot\Godot_v4.7-stable_win64_console.exe --headless --path . -- --audit
```
«Проигрывает» мир: BFS-проходимость всех локаций, зачищаемость этажей шахты,
выполнимость квестов, целостность опций NPC, симметрию переходов — должен печатать
`НАЙДЕНО ПРОБЛЕМ: 0`. **Гоняй тесты И аудит после правок контента** — аудит ловит
битые данные, которые тесты не видят.

## Скриншотер (`tests/shot.gd`) — как «увидеть» игру без рук

`Main` по флагу `--shot` грузит `Shot.tscn`, который рендерит ключевые экраны в PNG
(в `user://` или заданную папку) и выходит. Запуск — **оконный** (headless не рисует):
```
tools\godot\Godot_v4.7-stable_win64_console.exe --path . -- --shot
```
Полезно, чтобы ИИ мог посмотреть на результат правок вёрстки/спрайтов. Владелец
разрешил открывать окно для отладки (скриншотер сам закрывается); всё равно
добавляй `--quit-after` как страховку. Логику тестируй headless — окно только
для визуальной проверки.

## Связь со старым проектом и сидер (`tools/seed.py`)

Данные `data/*.json` изначально **сгенерированы** из pygame-версии (`../burmolda-python`)
скриптом `tools/seed.py` (Python 3, только stdlib). Он импортирует движко-независимые
модули оригинала (предметы, локации, NPC, реплики, мобы, спрайты) и дампит в JSON.

Сейчас **источник истины — сам JSON** (правь его). Сидер нужен, только если захочешь
что-то заново вытащить из оригинала:
```
python tools\seed.py
```
⚠️ Перезапуск сидера **перезапишет** `data/*.json` — потеряешь ручные правки. Обычно он
больше не нужен.
